# Dispare v4.1 — Hardening. Инструкция по установке

Обновление защищает деньги и данные. Ничего не переписывается, база **не пересоздаётся**
(новая таблица `emex_order_log` создаётся сама). Все изменения обратимы.

**Что входит:**
1. Пароль → Argon2id (старый пароль продолжит работать и обновится сам при входе).
2. Убраны небезопасные значения по умолчанию (`SECRET_KEY`, пароль `admin`).
3. Защита от повторной загрузки заказа Emex (двойного списания склада).
4. Резервное копирование базы (правильное, для WAL).
5. `/health` + healthcheck в Docker.
6. Скрипты `deploy.sh` / `rollback.sh`.

Порядок важен. Делайте по шагам, не пропуская. Всё — в PuTTY под root, из папки проекта:

```
cd /root/dispare-tool/dispare-tool-v3
```

---

## Шаг 0. Резервная копия (ДО всего)

Пока стоит старая версия — делаем корректную горячую копию базы:

```
docker compose exec app python -c "import sqlite3; s=sqlite3.connect('data/dispare.db'); d=sqlite3.connect('data/backup_before_v4.db'); s.backup(d); d.close(); s.close(); print('backup ok')"
```

Должно напечатать `backup ok`. Копия лежит в `data/backup_before_v4.db` (на хосте, переживёт пересборку).

---

## Шаг 1. Прописать SECRET_KEY в .env

Новая версия **не запустится** без `SECRET_KEY` — это защита от случайного старта с дефолтом.

Сгенерировать ключ:
```
openssl rand -hex 32
```
Скопируйте вывод (64 символа). Откройте `.env`:
```
nano .env
```
Убедитесь, что есть строка (вставьте свой ключ вместо ...):
```
SECRET_KEY=вставьте_сюда_64_символа
```
Если строка `SECRET_KEY=` уже была — **замените** значение на новое. Сохранить: `Ctrl+O`, `Enter`, выйти `Ctrl+X`.

> После смены SECRET_KEY все текущие сессии разлогинятся — это нормально, войдёте заново.

`ADMIN_PASSWORD` пока не трогайте (удалим на шаге 5).

## Шаг 2. Забрать новый код

Сначала залейте новые файлы в GitHub (замена в вашем локальном репозитории → commit → push), затем на сервере:
```
git pull
```

## Шаг 3. Сборка и запуск

Автоматически (рекомендуется):
```
bash deploy.sh
```
Скрипт сам сделает бэкап, соберёт, запустит и проверит `/health`. Если увидите строку
**«ГОТОВО. Приложение живо, /health = 200»** — всё поднялось.

> Вручную, если предпочитаете: `docker compose build --no-cache app && docker compose up -d`

---

## Шаг 4. Обновить базу (журнал заказов + защита от повтора)

```
docker compose exec app python -m app.migrate
```
Занесёт уже обработанные заказы (№1, 2, 4, 5, 6) в журнал, чтобы повторная загрузка
их старых файлов тоже блокировалась. Запускать можно повторно — лишнего не сделает.

---

## Шаг 5. Сменить пароль (правильно)

```
docker compose exec app python -m app.set_password
```
Введёте новый пароль дважды (на экране он не отображается). В базе останется только
необратимый Argon2id-хеш. Пароль минимум 8 символов.

Теперь уберите пароль из `.env` — он больше не нужен:
```
nano .env
```
Удалите (или закомментируйте `#`) строку `ADMIN_PASSWORD=...`. Сохранить, выйти.
Применить:
```
docker compose up -d
```

---

## Шаг 6. Проверка

- [ ] `curl http://localhost/health` → `{"status":"ok","database":"ok"}`
- [ ] Вход по **новому** паролю работает (обновите Ctrl+F5)
- [ ] Вход по **старому** паролю `Dispare2024` больше **не** работает
- [ ] Загрузите один и тот же файл заказа дважды → второй раз появится красная плашка
      «Заказ №… уже обрабатывался», склад **не** спишется повторно
- [ ] Настройте ежедневный бэкап (см. ниже)

---

## Ежедневный бэкап (одноразовая настройка)

Добавить в cron ежедневную копию в 03:00:
```
crontab -e
```
Вставьте строку (путь под ваш сервер):
```
0 3 * * * cd /root/dispare-tool/dispare-tool-v3 && docker compose exec -T app python -m app.backup_db >> /root/dispare-backup.log 2>&1
```
Копии складываются в `data/backups/`, хранятся последние 30. **Рекомендация:** периодически
копируйте эту папку на внешний диск/облако — том на том же сервере это ещё не полноценный бэкап.

---

## Если что-то сломалось

Откат кода на предыдущую версию:
```
bash rollback.sh
```
Если новая версия успела испортить данные — восстановите базу из копии:
```
docker compose down
cp data/backup_before_v4.db data/dispare.db
docker compose up -d
```

## Чего здесь СОЗНАТЕЛЬНО нет (отдельными шагами позже)

- **HTTPS** — у вас уже готов пакет `dispare-markup-https.zip`, разворачиваем его отдельно.
- **Полная сущность EmexOrder** (перенос номера заказа из notes в таблицу) — большой рефактор
  на живой базе, пока не нужен: защиту от повтора мы сделали без него.
- **Audit Log во все действия** — полезно для инвестора, но трогает много мест; сделаем отдельно.
- **PostgreSQL / ФИФО** — не сейчас, для вашего масштаба SQLite+WAL достаточно.

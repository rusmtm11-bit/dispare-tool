# Dispare Trading — Инструмент проценки

Веб-приложение для анализа цен на автозапчасти: сравнение закупочных цен от поставщиков с рыночными ценами (Emex), три вида ваших продажных цен, история, мониторинг.

---

## Пошаговая инструкция: от нуля до работающего сервиса

### Шаг 1. Купить VPS на reg.ru

1. Зайдите на https://www.regru.ru → раздел «VPS» (или «Серверы»)
2. Выберите тариф **VDS SSD-2** (2 ГБ RAM, 40 ГБ диск) — примерно 990 ₽/мес
   - Если есть акция на бесплатный период — берите
3. ОС при заказе: выберите **Ubuntu 22.04** или **Ubuntu 24.04**
4. После оплаты вы получите на почту:
   - **IP-адрес** сервера (например, 194.67.xx.xx)
   - **Логин** (обычно `root`)
   - **Пароль**

Запишите эти данные — они понадобятся далее.

---

### Шаг 2. Направить поддомен на VPS

1. Зайдите в личный кабинет reg.ru → ваш домен `disparetrading.ru` → **DNS-записи**
2. Нажмите «Добавить запись»
3. Заполните:
   - Тип: **A**
   - Имя поддомена: **app**
   - IP-адрес: вставьте IP вашего VPS
4. Сохраните. DNS обновится за 5–30 минут.

Теперь `app.disparetrading.ru` будет вести на ваш VPS.

---

### Шаг 3. Подключиться к серверу

**На Windows:**
1. Скачайте программу [PuTTY](https://www.putty.org/) или используйте встроенный терминал Windows (cmd)
2. В cmd наберите: `ssh root@ВАШ_IP_АДРЕС`
3. Введите пароль из письма reg.ru

**На Mac:**
1. Откройте «Терминал» (Applications → Utilities → Terminal)
2. Наберите: `ssh root@ВАШ_IP_АДРЕС`
3. Введите пароль

Вы увидите командную строку сервера — значит, всё работает.

---

### Шаг 4. Установить Docker на сервер

Скопируйте эти команды **по одной** и вставьте в терминал, нажимая Enter:

```bash
# 1. Обновить систему
apt update && apt upgrade -y

# 2. Установить Docker
curl -fsSL https://get.docker.com | sh

# 3. Установить Docker Compose
apt install -y docker-compose-plugin

# 4. Проверить, что всё работает
docker --version
docker compose version
```

Если обе команды показали версии — Docker установлен.

---

### Шаг 5. Скачать проект на сервер

```bash
# 1. Установить git (если не установлен)
apt install -y git

# 2. Перейти в домашнюю папку
cd /root

# 3. Создать папку и скопировать файлы проекта
# ВАРИАНТ А: Если вы загрузили проект в свой GitHub-репозиторий:
git clone https://github.com/rusmtm11-bit/disparetrading-tool.git dispare-tool

# ВАРИАНТ Б: Если файлы у вас на компьютере — загрузите их через SCP:
# (выполните на СВОЁМ компьютере, не на сервере)
# scp -r /путь/к/dispare-tool root@ВАШ_IP:/root/dispare-tool

cd dispare-tool
```

---

### Шаг 6. Настроить конфигурацию

```bash
# 1. Скопировать пример настроек
cp .env.example .env

# 2. Открыть файл для редактирования
nano .env
```

В открывшемся редакторе:
- **SECRET_KEY**: замените на любую длинную случайную строку (можно набить по клавиатуре 40+ символов)
- **ADMIN_PASSWORD**: придумайте свой пароль для входа
- **EMEX_API_KEY**: пока оставьте пустым (подключите позже)
- Остальное можно не трогать

Чтобы сохранить: нажмите `Ctrl+O`, потом `Enter`, потом `Ctrl+X`.

---

### Шаг 7. Запустить приложение

```bash
# Создать папку для данных
mkdir -p data

# Запустить всё одной командой
docker compose up -d --build
```

Подождите 1–2 минуты, пока всё соберётся. Потом проверьте:

```bash
docker compose ps
```

Должно быть два контейнера в статусе `Up`: `dispare-app` и `dispare-nginx`.

---

### Шаг 8. Проверить, что работает

Откройте в браузере: **http://app.disparetrading.ru**

Вы увидите страницу входа. Введите:
- Логин: `admin` (или что указали в .env)
- Пароль: что указали в .env

Если страница открылась — всё работает!

---

### Шаг 9 (опционально). Включить HTTPS

Это нужно для безопасности (пароли передаются в зашифрованном виде):

```bash
# 1. Получить SSL-сертификат (замените email на свой)
docker compose run certbot certonly --webroot -w /var/www/certbot -d app.disparetrading.ru --email ваш@email.com --agree-tos --no-eff-email

# 2. Отредактировать nginx.conf — раскомментировать блок ЭТАП 2
nano nginx.conf
# Удалите символы # перед строками в секции "ЭТАП 2"
# И закомментируйте (добавьте #) секцию "ЭТАП 1"

# 3. Перезапустить nginx
docker compose restart nginx
```

Теперь `https://app.disparetrading.ru` работает с шифрованием.

## Как пользоваться

### Первый запуск — загрузка каталога поставщика

1. Зайдите в раздел **Каталог**
2. Добавьте поставщика (название + валюта)
3. Загрузите его Excel-прайс — система автоматически найдёт колонки с артикулами, ценами и описаниями
4. Повторите для каждого поставщика

### Загрузка ваших цен

1. Зайдите в раздел **Наши цены**
2. Загрузите Excel: первая колонка — артикул, далее три колонки цен (заказ / 3PL / 3PL+Emex)
3. Или введите цены вручную по одной

### Поиск и сравнение

1. Зайдите в **Поиск**
2. Введите артикул → увидите: ваши 3 цены, закупочные цены поставщиков (с конвертацией в рубли), рыночные предложения из Emex
3. Для массового сравнения — вкладка «Список» или «Загрузить файл»
4. Галочка «Запросить Emex» — запрашивает свежие данные (платно, 0.12 ₽ за артикул)

### Мониторинг

1. Раздел **Мониторинг** — добавьте артикулы для автоматического отслеживания
2. Система будет обновлять цены 1–2 раза в день по расписанию
3. В разделе **Обзор** — бюджет Emex (сколько потрачено за день/месяц)

### Добавление коллег

1. Зайдите в **Настройки** (видно только админу)
2. Создайте аккаунт для коллеги — логин и пароль
3. Дайте ему ссылку `https://app.disparetrading.ru`

## Подключение Emex API

Когда будете готовы подключить реальный Emex:

1. Зарегистрируйте IP вашего сервера в Emex (через поддержку или ваш ПВЗ)
2. Получите API-ключ
3. Пропишите его в .env: `EMEX_API_KEY=ваш-ключ`
4. Перезапустите: `docker compose restart app`

Возможно, формат ответа вашего API отличается от заглушки — в этом случае нужно будет адаптировать файл `app/services/emex_client.py` (функция `fetch_emex_prices`). Напишите мне с примером ответа от Emex, и я подправлю.
